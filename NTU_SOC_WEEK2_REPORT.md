# NTU SoC Week 2 Report

## Week 2 Objective

Implement source citations for responses generated by the PDF chatbot so that users can identify the originating document and page number for each answer.

---

## Work Completed

### 1. Feature Investigation

* Studied the existing document ingestion and retrieval pipeline.
* Investigated how LangChain stores document metadata.
* Explored different approaches for preserving metadata through text chunking and vector storage.
* Identified the changes required to support source citations without disrupting the existing chatbot functionality.

### 2. Feature Design

* Prepared a feature design document outlining the implementation strategy.
* Planned modifications to the document ingestion, chunking, vector store creation, retrieval pipeline, and user interface.
* Decided to preserve metadata using LangChain `Document` objects.

### 3. Metadata Preservation

Refactored the PDF processing pipeline to preserve document metadata.

Changes included:

* Creating `Document` objects instead of returning raw text.
* Storing:

  * PDF filename
  * Page number
* Ensuring metadata remained attached throughout the processing pipeline.

### 4. Document Chunking

Modified the chunking process to use `split_documents()` instead of `split_text()` so that metadata was retained for every generated chunk.

### 5. Vector Store Update

Updated FAISS vector store creation to use `FAISS.from_documents()` rather than `FAISS.from_texts()`.

This ensured that metadata was indexed together with document content.

### 6. Retrieval Chain Enhancement

Updated the conversational retrieval chain to:

* Return source documents along with answers.
* Configure conversation memory to support multiple output fields using `output_key="answer"`.

### 7. Source Citation Interface

Implemented a citation display beneath every chatbot response.

The interface displays:

* PDF filename
* Page number(s)

Additional improvements:

* Removed duplicate citations.
* Grouped multiple page references from the same PDF into a single entry for improved readability.

---

## Testing

Performed testing using:

* Single PDF uploads
* Multiple PDF uploads

Verified:

* Correct answer generation.
* Metadata preservation throughout the retrieval pipeline.
* Display of source filenames.
* Display of page numbers.
* Grouping of page references for cleaner presentation.

Observed that retrieval quality occasionally included unrelated documents due to similarity search behaviour. This is a retrieval limitation rather than an issue with the citation implementation.

---

## Challenges Encountered

### LangChain Memory Error

Encountered a compatibility issue after enabling source document retrieval:

```
ValueError: One output key expected
```

Resolved by explicitly specifying:

```python
output_key="answer"
```

within `ConversationBufferMemory`.

### Metadata Preservation

Initially lost metadata during chunking because raw text was used.

Resolved by switching to LangChain `Document` objects and using `split_documents()`.

### Retrieval Behaviour

Observed that similarity search occasionally returned chunks from unrelated PDFs.

Improved citation readability by:

* grouping pages by document,
* reducing duplicate citations,
* tuning retrieval parameters.
